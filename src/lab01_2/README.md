
###Вопрос 1. Что является сущностью?

####В предметной области «Игровая логика» существует несколько возможных сущностей:

игрок, персонаж, оружие, предмет инвентаря, квест. Центральным объектом, вокруг которого строится любая игровая система, является игрок (Player). Именно игрок взаимодействует с миром: получает урон, набирает опыт, повышает уровень, может погибнуть или быть заблокирован. Все остальные сущности — вторичны по отношению к нему.
     
###Вопрос 2. Какие у него атрибуты?

#### После анализа предметной области были выделены следующие характеристики (атрибуты) объекта Player:

АтрибутТипОписаниеnamestrУникальный игровой псевдоним игрокаlevelintТекущий уровень (от 1 до 100)healthfloatОчки здоровья (не может быть отрицательным)experienceintНакопленный опыт (не может быть отрицательным)statusstrЛогическое состояние: active, dead, banned
Также добавлен атрибут класса MAX_LEVEL = 100 — единое ограничение уровня для всех экземпляров. Это значение принадлежит классу, а не конкретному объекту.
     
####Поведение (бизнес-методы):

take_damage(damage) — получить урон, автоматически переходит в состояние dead при HP = 0
gain_experience(points) — набрать опыт с автоматическим повышением уровня

###Вопрос 3. Какие инварианты?

####Инварианты — правила, которые объект обязан соблюдать всегда:

name не может быть пустой строкой или пробелами
level находится в диапазоне от 1 до 100 включительно
health не может быть отрицательным числом
experience не может быть отрицательным числом
status принимает только значения из множества {"active", "dead", "banned"}
Нельзя повысить уровень выше MAX_LEVEL = 100
При получении летального урона (HP = 0) статус автоматически меняется на dead

Все инварианты вынесены в отдельный файл validate.py и проверяются как в конструкторе, так и в сеттерах.

###Вопрос 4. Что значит «равенство»?

Два объекта Player считаются равными, если совпадают их name и level. Это логично: в игровой системе именно комбинация имени и уровня однозначно идентифицирует игрока в рейтинге. Состояние здоровья, опыт и статус — изменяемые данные, которые не влияют на идентичность объекта.

###Вопрос 5. Есть ли состояние?

####Да. Объект Player имеет три чётко выраженных состояния:

СостояниеОписаниеОграниченияactiveНормальная играВсе действия доступныdeadИгрок погибНельзя получать урон, нельзя набирать опыт; доступен только respawn()bannedИгрок заблокированВсе действия заблокированы
Переходы между состояниями:

active → dead: автоматически при HP = 0
dead → active: через метод respawn()
active/dead → banned: через метод ban()

###Сценарий 1 — Создание объектов, магические методы, атрибут класса

Создаются три игрока. Выводится строковое представление через __str__ и отладочное через __repr__. Проверяется __eq__: два игрока с одинаковым именем и уровнем — равны, с разными — нет. Демонстрируется доступ к атрибуту класса MAX_LEVEL как через класс, так и через экземпляр.

###Сценарий 2 — Бизнес-логика: урон, опыт, повышение уровня

Игрок получает опыт порциями. Первая порция не превышает порог уровня — уровень не растёт. Вторая порция пересекает порог — происходит level up. Затем игрок получает урон дважды. Через сеттер health напрямую устанавливается новое значение здоровья (лечение).

 ###Сценарий 3 — Логическое состояние: смерть и возрождение
 
Игрок с низким HP получает летальный урон. Статус автоматически меняется на dead. Попытка набрать опыт в мёртвом состоянии блокируется с исключением RuntimeError. После вызова respawn() здоровье восстанавливается, статус возвращается в active, и игрок снова может действовать.

###Сценарий 4 — Логическое состояние: бан

Игрок получает статус banned. Все попытки взаимодействия — получить урон, набрать опыт, возродиться — блокируются. Это демонстрирует поведение, зависящее от состояния объекта.
