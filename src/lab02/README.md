### Как я строилa коллекцию

#### Что такое контейнер и зачем он нужен?

В первой лабораторной я создалa класс Player, который описывает одного игрока. Но в реальной игре игроков много. Значит, нужно что-то, что будет хранить их всех вместе и управлять ими. Для этого я создалa класс PlayerList.

Идея простая: PlayerList — это как коробка, внутри которой хранится список объектов Player. У этой коробки есть методы — добавить игрока, удалить, найти, отсортировать и отфильтровать.

### Какие правила я установилa для коллекции?
Пока думалa над реализацией, определил несколько ограничений:

Добавлять можно только объекты типа Player — если передать число или строку, будет ошибка
Нельзя добавить одного и того же игрока дважды — дубликаты блокируются
Нельзя удалить игрока, которого нет в списке
Нельзя обратиться к индексу, которого не существует


Почему Model.py, collection.py и Demo2.py — это три разных файла?
Преподаватель объяснял, что архитектура — это разделение ответственности. Поэтому:

Model.py — только импортирует Player из первой лабораторной, больше ничего не делает
collection.py — только логика контейнера, без валидации и без сценариев
Demo2.py — только сценарии использования, без логики внутри

### Сценарий 1 — Добавление и удаление игроков
Создаются 4 игрока и добавляются в список. Выводится полный список. Затем один игрок удаляется и список выводится снова. Также демонстрируется, что нельзя добавить объект, который не является Player, и что нельзя добавить игрока, который уже есть в списке.


### Сценарий 2 — len, for и поиск
Показывается количество игроков через len(). Список перебирается через for с выводом каждого игрока. Выполняется поиск по имени — один игрок найден, другой не найден и возвращается None. Также показывается поиск по уровню, который возвращает новый список с результатами.


### Сценарий 3 — Индексация и remove_at
Демонстрируется обращение к конкретным игрокам через lista[0] и lista[1]. Игрок удаляется по индексу через remove_at() и результат выводится. Также показывается обращение к несуществующему индексу — ошибка корректно обрабатывается.

### Сценарий 4 — Сортировка коллекции
